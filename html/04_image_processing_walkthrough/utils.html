Utils function description (docstrings)

Description of the functions in the utils module.

image_processing_scripts.utils.create_config_dict(config_string)

Function to create the config dictionary from a string. Needed because ROS only supports strings.

Parameters

config_string (list) – String array, that contains all necessary information for locating different types of stones [stone_type, stone_id, pallet_id, camera_height, xml_path, visualize, position_format, operation_mode, save_images] Assertion: All instance str

Returns

config_dict – Converted string as dictionary. Keys: {“stone_type”, “stone_id”, “pallet_id”, “camera_height”, “xml_path”, “visualize”, “position_format”, “operation_mode”, “save_images”}

Return type

dict

image_processing_scripts.utils.read_config_string_from_main()

Function to create config string from global variables in main. Used for development purposes only. Config string has to be created dynamically by coordinator (robot).

Returns

config_string – String array, that contains all necessary information for locating different types of stones [stone_type, stone_id, pallet_id, camera_height, xml_path, visualize, position_format, operation_mode, save_images] Assertion: All instance str

Return type

list

image_processing_scripts.utils.encode_stone_location(array)

Function to flatten results in case multiple stones are located. ROS util because action server/client only support string[]. An integer value to decode results later is appened to the end.

Parameters

array (list) – 2D array containing stone location either in format xyzz or homogenous

Returns

array_encoded – 1D array containing flattened input array and an int value to later decode.

Return type

list

image_processing_scripts.utils.decode_stone_location(array_encoded)

Function to decode results in case multiple stones are located. ROS util because action server/client only support string[]. An integer value to decode results ar input[-1]

Parameters

array_encoded (list) – 1D array containing an int value to decode.

Returns

array_decoded – 2D array containing stone location either in format xyzz or homogenous

Return type

list

image_processing_scripts.utils.get_homogeneous_coordinates_from_stone_location(stone_locations)

Function to convert from xyz, rotz stone location to homogenous coordinates

Parameters

stone_locations (list) – [[float, float, float, float],[float, float, float, float], …] Sequence of stone coordinate [x, y, z, rot_Z] in millimeter/rad

Returns

homogeneous_coordinates – Matrix 4x4 [float] Stone center location as homogenous matrix in millimeters/rad

Return type

list
