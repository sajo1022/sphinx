Main module

The main module in this project acts like a control panel for the whole image processing procedure. Every parameter which is or might be needed for a certain task can be set globally. The following gives a brief overview of the parameter and there potential values.

General parameters:

OPERATION_MODE:

This parameter controls whether the process should only acquire new images without processing them, acquire new images with processing or read local images and process them.

'save_only', 'acquire_images', 'read_images'

SAVE_IMAGES and SAVE_PATH:

This parameter sets whether the acquired images should be saved in a specific directory.

'True', 'False', '{PATH}'

VISUALIZE:

Sets a visualization mode. If the parameter is set to ‘debug’, almost every step of the image processing is displayed to find potential errors. If set to ‘step_by_step’, major interim results are visualized. This mode is well suited to showcase the method of the image processing. If set to ‘result’, only the outcome is displayed. Parameter value ‘no’ does not display anything.

'debug', 'step_by_step', 'result', 'no'

POSITION_FORMAT:

This parameter defines the format of the returned stone pose. If set to ‘xyzz’ the pose is returned as the xyz coordinates and the rotation around the z-axis. The parameter value ‘homogenous’ causes the return to be in homogenous coordinates.

'xyzz', 'homogenous'

CAMERA_HEIGHT:

Sets the camera height from the ground to get a clean background segmentation of the pallet.

{CAMERA_HEIGHT}

XML_PATH:

In order to process fitting stones, a xml file with the stones geometries is needed. The path determines where to find this xml file.

{PATH}

Stone related parameters:

STONE_TYPE:

Determines the stone type that should be processed.

'standard_stone', 'fitting_stone'

PALLET_ID:

Determines the id of the pallet on which the searched stone is positioned.

{PALLET_ID}

STONE_ID:

Determines the id of the searched stone.

{STONE_ID}
